# Introduction

Selecting an appropriate dataset is one of the most important decisions in this research project. Since the objective of this work is to investigate AI-assisted Detection Engineering for Security Operations Centers (SOCs), the dataset should represent realistic SOC investigations rather than synthetic attack simulations.

After exploring several publicly available cybersecurity datasets, the Microsoft GUIDE dataset was selected because it contains real-world security incidents collected from operational environments. Unlike traditional intrusion detection datasets, GUIDE represents the complete investigation process followed by SOC analysts, making it highly relevant for this research.

---

# Dataset Overview

The GUIDE dataset is currently the largest publicly available collection of real-world security incidents released by Microsoft for cybersecurity research.

The dataset contains:

| Attribute | Value |
|-----------|-------|
| Organizations | More than **6,100** |
| Security Incidents | Approximately **1 million** |
| Security Alerts | Approximately **1.6 million** |
| Evidence Records | More than **13 million** |
| Entity Types | **33** |
| Detector IDs | Approximately **9,100** |
| MITRE ATT&CK Techniques | **441** |
| Time Period | Two weeks of production telemetry |

Unlike many academic datasets, GUIDE contains analyst-labelled incidents collected from operational SOC environments, making it suitable for studying real-world detection and investigation workflows. :contentReference[oaicite:0]{index=0}

# Why This Dataset?

Several reasons influenced the selection of this dataset.

First, the data originates from real operational environments rather than laboratory simulations. This makes the alerts and incidents more representative of the challenges faced by modern SOC analysts.

Second, the dataset contains complete security incidents instead of isolated log entries. This allows the investigation process to be studied rather than focusing only on anomaly detection.

Another important advantage is the availability of analyst-provided triage labels, enabling the evaluation of AI-generated recommendations against real analyst decisions. :contentReference[oaicite:1]{index=1}

# Dataset Structure

The GUIDE dataset is organized around three main concepts.

## Incident

An incident represents a complete security investigation. It may consist of multiple related alerts that are grouped together by analysts.

Example:

```
Incident

↓

PowerShell Execution

↓

Credential Dumping

↓

Suspicious Login

↓

Data Exfiltration
```

---

## Alert

Each incident contains one or more alerts generated by security products.

Examples include:

- Suspicious PowerShell execution
- Malicious process creation
- Network anomaly
- Suspicious authentication
- Malware detection

Alerts form the primary input for SOC investigations.

---

## Evidence

Each alert contains multiple pieces of supporting evidence.

Examples include:

- User accounts
- Devices
- Files
- IP addresses
- URLs
- Registry keys
- Processes

These evidence records provide the context required to understand the incident.

# Strengths

The GUIDE dataset offers several advantages for this research.

It is based on real-world SOC telemetry collected from thousands of organizations, providing realistic security scenarios that are difficult to reproduce in laboratory environments.

The inclusion of analyst-labelled incidents makes it valuable for evaluating AI-assisted investigation workflows. In addition, the dataset covers hundreds of MITRE ATT&CK techniques, making it suitable for studying attack behaviour and contextual reasoning.

Another significant advantage is its scale, allowing experiments to be performed on a diverse range of incident types. :contentReference[oaicite:2]{index=2}

---

# Limitations

Although GUIDE is one of the most comprehensive cybersecurity datasets currently available, several limitations should be considered.

The dataset does not contain complete raw operating system logs. Instead, it provides processed alerts and incident information generated by Microsoft's security products.

As a result, it cannot replace a SIEM environment such as Wazuh for log collection or attack simulation. It should instead be viewed as an evaluation dataset for validating AI-assisted investigation techniques.
